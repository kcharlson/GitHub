Your HTML document must always start with the tag. This tag acts as a set of instruction to the web browser about what version of HTML the page is written in. Once you establish your tag, that’s when the core structure begins.

Start with your HTML tags <html></html>, everything in between those tags will be considered a part of your document. Look at that; you just made your first tag pairing. Once you understand how tags surround (nearly) everything in tag pairings, it’s easy to comprehend and write basic HTML.

A common misconception about the HTML tag is that everything in between the tag pairing will be visible. We will discuss the visual aspect of HTML very soon.

So now that you have a good start let’s focus on adding some details by adding a <head></head> section. The head section is filled with all the details that the browser will use to describe your website to search engines such as Google. This is where your site title, description, and keywords live. Also, the head section will be where you will content external documents such as your CSS stylesheet.

Just like HTML with the HTML tag, none of this info will be visible on your website, but be aware that be visible on search engines and preview windows.

We have already gone over the beginning aspects of the structure of HTML, but there’s a bit more to it. Within the body section, you have some tools to help create even more structure. The <header><nav><article><section><footer> tags can be used to help define certain parts of a website. For example, the <nav> would set the content related to the site’s navigation. Unlike say the <body> tag, these tags are not required for the site to function, but it is standard to use them because they help so much with the organization of the code.

Ah the body tag, this is where the magic happens. Everything placed within this set of tags will help build your website. In here you can place as much text, links, images, and tags as you want. As you enter information into the body of the document it will fill the browser window from top to bottom in the order you place the elements in the HTML document, simple right? As can see, all the items are entering the flow of the page hugging the left side of the page, in its default styling.

Of course, if you are going to build something cool for the web, you’re going to need some tools. We have already started to discuss how to use tags, but what tags are available to you as a designer? What’s in your toolkit?

These tags hold a lot of creative power and are the most common tags you will use so try to get as comfortable with them as possible. You can practice using these tags and see them in context by following the Codepen link.
